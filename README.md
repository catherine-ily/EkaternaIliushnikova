## Что внутри

Внутри каждой папки с блоком проектов лежит отдельный README файл с навигацией по проектам внутри нее.

### 1. Бизнес-анализ (BSA) (`/BSA`)
**Что представлено:** Учебные задания по анализу процессов.  
**Примеры задач:**
- Моделирование требований к бизнес-процессу: построение диаграммы потоков данных (DFD), межфункциональной диаграммы (swimlane), диаграммы состояний и таблицы решений для формализации и визуализации бизнес-логики.
- Анализ и доработка бизнес-процессов с помощью BPMN: построение BPMN-диаграмм, сравнение с ранее разработанными моделями (DFD, swimlane), выявление нестыковок и их устранение.
- Разработка пользовательских историй (User Stories) и критериев приёмки: создание US с критериями приемки и тестовыми сценариями, обсуждение с командой разработки для согласования требований.
- Описание требований через Use Case (варианты использования): разработка сценариев взаимодействия пользователей с системой, моделирование поведения программной системы для формализации функциональных требований.

**Навыки:** построение диаграмм, формализация процессов, написание требований.

### 2. SQL / PostgreSQL (`/SQL`)

**Что представлено:** Аналитические запросы, агрегации, оконные функции, CTE.  
**Примеры задач:**
- Классическая задача коммивояжёра (поиск самого бюджетного маршрута).
- Разработка ETL-процесса для хранилища данных (DWH).
- Разработка схемы БД для системы лояльности: ограничения, индексы, последовательности, массовые операции и документация.

**Навыки:** написание сложных запросов, работа с временными таблицами, подзапросы.

### 3. Язык C (`/C`)
**Что представлено:** Учебные алгоритмы и структуры данных.  
**Примеры задач:**
- Разработка утилит Bash по работе с текстом: cat, grep.
- Реализация библиотеки string.h.
- Реализация библиотеки s21_decimal.h.
- Реализация библиотеки matrix.h.

**Навыки:** алгоритмическое мышление, работа с памятью, обработка данных.
